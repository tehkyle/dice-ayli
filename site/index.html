First launch

1

Download & move to Applications

Open the .dmg, drag Dacha DICE: AYLI into your Applications folder, then eject the disk image.

2

Bypass Gatekeeper

The app is not notarized by Apple, so macOS will warn on first open. Right-click the app icon → Open → click Open in the dialog. You only need to do this once.

If macOS says the app is "damaged" instead of showing an Open button, run this in Terminal, then open normally:

xattr -dr com.apple.quarantine "/Applications/Dacha-Dice-AYLI.app"
3

Connect Google Sheets

Click the gear icon in the top-right corner of the welcome screen. Connect your Google account, pick the spreadsheet, and choose the tab to write show reports to. The gear badge turns green when configured.

Show day

1

Open the app & check QLab

The welcome screen shows a QLab status indicator. Wait for it to turn green — this confirms the app has connected to the QLab workspace. If it stays orange, check that QLab is open and on the same machine.

2

Begin cast entry

Click Begin Cast Entry. The app shows today's date and performance number automatically.

3

Select scenes

Check which scenes are running in each act. Use Fix Order to set a specific sequence; leave it off for random order. Grayed-out scenes are not available to select.

4

Assign cast

Use the dropdowns to assign an actor to each character track. The app prevents duplicate assignments and highlights conflicts in red. Use Randomize unfilled if needed.

5

Lock & send to QLab

Review the summary, then click Lock & Send to QLab. The app writes actor names into QLab's memo cues and fires the CAST_CONFIRMED cue. A green banner confirms QLab was notified.

6

After the show

When QLab fires the show-end cue, Dacha DICE: AYLI automatically appends a row to the configured Google Sheet with the date, performance number, start time, run time, full cast, and scenes played in each act.

Show history

Every performance is saved locally. Click View show history → on the welcome screen to browse past shows. You can correct cast assignments after the fact using the edit button on any card.

Troubleshooting

QLab status stays orange
Make sure QLab is open, the workspace name in config.json matches, and the passcode is correct.
Google Sheets badge stays red
Open the gear icon and reconnect your Google account. Check that GOOGLE_OAUTH_CLIENT_ID and GOOGLE_OAUTH_CLIENT_SECRET are set in the app's .env.
Row not appearing in the sheet
Check that the signed-in Google account has Editor access to the spreadsheet. Also confirm the selected tab name hasn't been renamed.
App won't open on macOS
Right-click the app icon → Open. If System Settings blocks it, go to Privacy & Security and click Open Anyway.

Technical setup for production team

QLab requirements

The workspace must have the following memo cues. The app targets them by cue number — names don't matter.

Cue numberPurpose
Track_1 – Track_6Stores actor name per character track
A2_SCENESStores available scene IDs for Act 2
A3_SCENESStores available scene IDs for Act 3
A4_SCENESStores available scene IDs for Act 4
CAST_CONFIRMEDFired after cast is locked — triggers show start

OSC: app sends to port 53000, receives on 53001. Set workspace name and passcode in config.json.

Google Sheets setup

You need a Google Cloud project with the Sheets and Drive APIs enabled, and an OAuth2 client. Full instructions are in GOOGLE_SETUP.md in the repo.

The Client ID and Secret go in the app's .env file (or as GitHub Secrets if building from source). Everything else — which account, which spreadsheet, which tab — is configured at runtime via the gear icon.

The Google Sheet row format is:
Date · Perf # · Start · Run time · [cast per track] · [scenes per act]
